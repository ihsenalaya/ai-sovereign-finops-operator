## 1. Objectif scientifique

Ne pas présenter le papier comme :

> “Nous avons construit un opérateur Kubernetes.”

Le papier doit être formulé comme une contribution scientifique :

> **Economic-Aware and Sovereignty-Constrained Routing for Enterprise LLM Gateways**

L’opérateur Kubernetes est l’artefact expérimental qui implémente et valide l’approche.

## 2. Problème de recherche

Les entreprises utilisent de plus en plus les LLMs, mais elles ne savent pas :

* quel modèle coûte réellement le plus
* quelle équipe consomme quoi
* quand router vers un modèle moins cher
* quel est le coût réel d’une contrainte de souveraineté
* quand l’auto-hébergement GPU devient plus rentable qu’une API managée
* comment maintenir le service quand le budget est presque consommé

Le but scientifique est de montrer qu’un plan de contrôle placé au niveau de la gateway Envoy peut réduire les coûts LLM tout en conservant qualité, latence acceptable et conformité aux politiques de souveraineté.

## 3. Hypothèse centrale

Un plan de contrôle LLM basé sur Envoy, enrichi par des signaux FinOps et souveraineté, peut réduire le coût opérationnel des applications IA sans dégradation significative de qualité de réponse ni violation des politiques déclaratives.

## 4. Questions de recherche

Préparer le protocole expérimental autour des RQ suivantes.

### RQ1 — Cost Reduction

Question :

> Can economic-aware routing reduce LLM operational costs compared with static routing strategies?

Comparer :

* Baseline 1 : always GPT-4o / modèle premium
* Baseline 2 : round-robin
* Baseline 3 : least-cost model
* Baseline 4 : static model mapping
* Notre approche : economic-aware routing via Envoy + operator

Métriques :

* coût total
* coût par requête
* coût par token
* coût par équipe
* coût par application
* économie relative en %

### RQ2 — Quality Preservation

Question :

> Does cost-aware routing preserve response quality within an acceptable degradation threshold?

Comparer la qualité entre :

* modèle premium fixe
* routage économique
* routage low-cost naïf

Métriques possibles :

* exact match si dataset QA
* F1 score si extraction
* LLM-as-a-Judge
* win-rate pairwise
* taux de réponses acceptables
* taux d’échec fonctionnel

Objectif attendu :

* réduction coût significative
* perte qualité inférieure à 3–5 %

### RQ3 — Latency and Reliability

Question :

> What is the operational overhead of gateway-level economic routing?

Mesurer :

* latence p50 / p95 / p99
* throughput
* taux d’erreur
* surcharge CPU/mémoire côté Envoy/operator
* temps de décision du routing
* disponibilité du service

Comparer :

* Envoy seul
* Envoy + observabilité
* Envoy + operator + routing dynamique

### RQ4 — Sovereignty Constraints

Question :

> What is the cost and performance impact of enforcing sovereignty constraints on LLM routing?

Scénarios :

* Global unrestricted
* EU only
* France only
* no external provider for sensitive data
* self-hosted only

Métriques :

* coût supplémentaire lié à la souveraineté
* violations détectées
* violations évitées
* latence supplémentaire
* impact qualité
* nombre de requêtes reroutées

### RQ5 — Budget-Aware Graceful Degradation

Question :

> Can budget-aware degradation maintain service availability compared with hard blocking policies?

Comparer :

* hard block quand budget atteint
* alert only
* graceful degradation : modèle premium → modèle moyen → modèle local/cache

Métriques :

* disponibilité
* coût
* qualité
* taux de requêtes servies
* taux de requêtes bloquées
* budget overrun

### RQ6 — Managed API vs Self-Hosted Break-even

Question :

> Can runtime telemetry predict when self-hosting becomes economically preferable to managed LLM APIs?

Faire varier :

* volume tokens/jour
* coût GPU
* coût ops
* coût migration
* modèle utilisé
* taux d’utilisation GPU
* batching
* cache hit ratio

Comparer :

* Azure OpenAI / OpenAI / Mistral API
* vLLM auto-hébergé sur GPU
* modèle hybride

Métriques :

* coût mensuel API
* coût mensuel self-hosted
* point mort en mois
* économie potentielle
* erreur de prédiction
* sensibilité aux hypothèses
